运维开发的本质是对由于运维复杂度提升导致资源分散的整合

运维开发概念

  • CMDB 运维配置数据库
    • 主机
    • 系统
    • 网络
    • 服务
    • 平台 API(主要用于各系统间的整合)
  • 多租户
    • 企业空间（原子单位）
      • 区域(规模大的企业还会引入区域的概念)
      • 项目
      • 角色
      • 用户
      • 管理员

  • 单租户（企业）

    • 项目
    • 角色
    • 用户
    • 管理员

  • 项目

    • 应用
      • 应用框架
      • 插件（插件接口是一个协议）
        • 通用插件接口
          • 初始化
          • 反初始化
          • 启动
          • 停止
        • 专用插件接口
        • 插件A
        • 插件B
      • 公共函数库
    • 选型
    • 源代码管理
    • 三方接口
    • 发布
    • 升级
  • 接口（定义协议）
  • 类 （定义功能）
    • 类<-接口->类（实现依赖，设计模式产生）
  • 方法 (定义实现)
  • 初始化方法 (对功能参数初始化)

  • 属性 (定义功能的参数，机器学习本质不停调整参数，使之达到收敛)

  • 抽象(抽象最简单的理解：把大象装冰箱需要几步？任何东西都可以这样抽象，再解构)

  • 系统

    • 部分A
    • 部分B …
    • 部分间相互依赖 （最复杂的依赖关系：笛卡尔集）
      • 强依赖（多次依赖）
      • 弱依赖（一次依赖）
  • 数据结构
  • 设计模式
